# M5 — Archive Replay + Spoiler Gating Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Let players replay past puzzles in a non-recorded "practice" mode launched from the History drawer, and stop the drawer from spoiling answers for days the player hasn't solved yet.

**Architecture:** Generalize the guess/solution server actions to accept an optional `date` (falling back to today's `daily_puzzle`), and add `getArchivePuzzle(date)`. Add `lib/solvedDates.ts` to track which dates the player has finished (locally). `GameInterface` gains a client-side practice mode: it swaps the active puzzle, resets game state, routes guesses with the archive date, and suppresses ALL recording/persistence. The History drawer gates answer spoilers behind solved dates and adds a PLAY button. Builds on M1–M4 (branch `feat/skinline-bonus`).

**Tech Stack:** Next.js 16, React 19, TypeScript, Vercel KV, vitest, Tailwind v4.

## Global Constraints

- Past puzzles are stored under `puzzle:{date}` KV keys (already used by `getPuzzleHistory`). Today's is `daily_puzzle`. No new infra.
- **Practice mode must never write anything server-side or to daily localStorage:** no `submitGameStats`, no `submitBonusSolved`, no `recordUserResult`, no streak, no `fusion_daily_status` writes.
- The daily game is unaffected when not in practice. Entering practice resets in-memory game state; **exiting practice does a full reload** (`window.location.reload()`) to cleanly restore the daily puzzle from its persisted state.
- Spoiler rule: the History drawer shows champion names + skin line ONLY for dates in `localStorage['fusion_solved_dates']`; other rows hide them and show a PLAY button. A date is added to that set whenever the player finishes that puzzle (win OR surrender), daily or practice.
- `getArchivePuzzle` only serves dates strictly before today and never returns answers.
- `npx tsc --noEmit` clean after each task; `npm test` green. Do NOT run `npm run build`.

- [ ] **Step 2: Add `getArchivePuzzle`**

```ts
export async function getArchivePuzzle(date: string): Promise<{ imageUrl: string; date: string } | null> {
  try {
    const today = new Date().toISOString().slice(0, 10);
    if (!date || date >= today) return null; // archive = strictly past
    const puzzle = await kv.get<DailyPuzzle>(`puzzle:${date}`);
    if (!puzzle) return null;
    return { imageUrl: puzzle.imageUrl, date: puzzle.date };
  } catch (error) {
    console.error('Failed to get archive puzzle:', error);
    return null;
  }
}
```

- [ ] **Step 1: Create `lib/solvedDates.ts`**

```ts
export const SOLVED_DATES_KEY = 'fusion_solved_dates';

export function getSolvedDates(): string[] {
  try {
    const raw = localStorage.getItem(SOLVED_DATES_KEY);
    const parsed = raw ? JSON.parse(raw) : [];
    return Array.isArray(parsed) ? parsed.filter((d) => typeof d === 'string') : [];
  } catch {
    return [];
  }
}

export function addSolvedDate(date: string): void {
  try {
    const set = new Set(getSolvedDates());
    set.add(date);
    localStorage.setItem(SOLVED_DATES_KEY, JSON.stringify([...set]));
  } catch {
    /* ignore */
  }
}

export function hasSolvedDate(date: string): boolean {
  return getSolvedDates().includes(date);
}
```
