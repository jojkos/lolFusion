# M4 — Hints + Partial Give-Up Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Give stuck players non-surrender options during the champion phase — a role hint, a "reveal more" de-zoom, and a partial "reveal a champion" — each at a score cost, all flowing through the existing scoring/win machinery.

**Architecture:** Add hint-cost constants to the pure `lib/scoring.ts`. Add a `getChampionHint` server action (returns a champion's role without leaking the name). In `GameInterface`, extract the win sequence into a single `finishWin(...)` helper so both a guessed second champion and a revealed second champion complete the game identically; track an accumulating `hintPenalty` that feeds `computeBaseScore`. UI: a hints row during phase 1. Builds on M1–M3 (branch `feat/skinline-bonus`).

**Tech Stack:** Next.js 16, React 19, TypeScript, Vercel KV, DDragon, vitest, Tailwind v4.

## Global Constraints

- Hint costs (in `lib/scoring.ts`): `HINT_ROLE_COST = 10`, `HINT_ZOOM_COST = 10`, `REVEAL_CHAMPION_COST = 30`. These accumulate into `hintPenalty`, which is passed as the 2nd arg to `computeBaseScore(champTries, hintPenalty)`. The score floor (10) still applies — penalties can't drive score negative.
- Hints/partial-reveal are available ONLY during `phase === 'phase1'`.
- A role hint is one-shot per champion slot. "Reveal more" de-zooms by 0.5 down to the 1.0 floor (same mechanic as a wrong guess) and is free of cost once already at the floor (disabled). "Reveal a champion" reveals one unsolved slot (no guess); if that completes both champions, the game is won (with the penalty applied).
- The role hint must NOT leak the champion name to the client — only the role/class string.
- `hintPenalty`, `hintsUsed`, and revealed role hints must persist to `localStorage` and restore on reload so the score stays correct.
- The recorded daily result's `hints` field (M3) must reflect the real `hintsUsed` count (was hard-coded 0).
- `npx tsc --noEmit` clean after each task; `npm test` green. Do NOT run `npm run build`. Server actions may use `fetch`/`new Date()`.

- [ ] **Step 5: Add `getChampionHint` to `app/actions.ts`**

```ts
export async function getChampionHint(slot: 'A' | 'B'): Promise<{ role: string } | null> {
  try {
    const puzzle = await kv.get<DailyPuzzle>('daily_puzzle');
    if (!puzzle) return null;
    const name = slot === 'A' ? puzzle.champA : puzzle.champB;
    const vRes = await fetch('https://ddragon.leagueoflegends.com/api/versions.json');
    const versions = await vRes.json();
    const latest = versions[0];
    const cRes = await fetch(
      `https://ddragon.leagueoflegends.com/cdn/${latest}/data/en_US/champion.json`,
    );
    const data = await cRes.json();
    const champ = Object.values(data.data).find(
      (c) => (c as { name?: string }).name?.toLowerCase() === name.toLowerCase(),
    ) as { tags?: string[] } | undefined;
    const role = champ?.tags?.[0] ?? 'Unknown';
    return { role };
  } catch (error) {
    console.error('Failed to get champion hint:', error);
    return null;
  }
}
```

  champTries: number,
  updatedRevealed: { A: string | null; B: string | null; Theme: string | null },

- [ ] **Step 3: `handleRoleHint`**

```ts
const handleRoleHint = async () => {
  const slot: 'A' | 'B' | null = !foundSlots.includes('A') && !roleHints.A
    ? 'A'
    : !foundSlots.includes('B') && !roleHints.B
    ? 'B'
    : null;
  if (!slot) return;
  const hint = await getChampionHint(slot);
  if (!hint) return;
  const newRoles = { ...roleHints, [slot]: hint.role };
  const newPenalty = hintPenalty + HINT_ROLE_COST;
  const newUsed = hintsUsed + 1;
  setRoleHints(newRoles);
  setHintPenalty(newPenalty);
  setHintsUsed(newUsed);
  setMessage({ ok: true, text: `Hint: ${slot === 'A' ? 'FIND · 1' : 'FIND · 2'} is a ${hint.role}.` });
  saveState({ roleHints: newRoles, hintPenalty: newPenalty, hintsUsed: newUsed });
};
```

- [ ] **Step 5: `handleRevealChampion` (partial give-up)**

```ts
const handleRevealChampion = async () => {
  const slot: 'A' | 'B' | null = !foundSlots.includes('A') ? 'A' : !foundSlots.includes('B') ? 'B' : null;
  if (!slot) return;
  const sol = await getSolution();
  if (!sol) return;
  const name = slot === 'A' ? sol.champA : sol.champB;
  const newFound = [...foundSlots, slot];
  const updatedRevealed = { ...revealedNames, [slot]: name };
  const newPenalty = hintPenalty + REVEAL_CHAMPION_COST;
  const newUsed = hintsUsed + 1;
  setFoundSlots(newFound);
  setRevealedNames(updatedRevealed);
  setHintPenalty(newPenalty);
  setHintsUsed(newUsed);
  hapticCorrect();
  if (newFound.includes('A') && newFound.includes('B')) {
    // Completing reveal wins the game (no guess → champTries unchanged = attempts).
    // Use the just-updated penalty for scoring.
    const score = computeBaseScore(attempts, newPenalty);
    setBaseScore(score);
    setPhase('won');
    setBonusStatus('open');
    setZoomLevel(1.0);
    triggerCelebration('win');
    hapticWin();
    await submitGameStats(attempts);
    fetchGlobalStats();
    recordDaily({ score, bonus: false, solved: true, givenUp: false, champTries: attempts });
    saveState({ foundSlots: newFound, phase: 'won', zoomLevel: 1.0, attempts, champTries: attempts, baseScore: score, bonusStatus: 'open', solved: true, revealedNames: updatedRevealed, hintPenalty: newPenalty, hintsUsed: newUsed });
    setTimeout(() => { if (!isMobile) resultsRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' }); }, 500);
  } else {
    setMessage({ ok: true, text: `Revealed: ${slot === 'A' ? 'FIND · 1' : 'FIND · 2'} is ${name}.` });
    saveState({ foundSlots: newFound, revealedNames: updatedRevealed, hintPenalty: newPenalty, hintsUsed: newUsed });
  }
};
```

(Note: this win branch uses `newPenalty` directly rather than the `hintPenalty` state, because `setHintPenalty` is async and `finishWin` would read the stale value. That's why this path is written inline rather than calling `finishWin`.)
